## üê≥ Docker Deployment

### Using Docker Compose (Recommended)

```bash
# Start all services
docker-compose up -d

# View logs
docker-compose logs -f

# Stop services
docker-compose down
```

### Manual Docker Commands

```bash
# MongoDB
docker run -d --name covid-mongodb -p 27017:27017 mongo:6

# Server
cd server
docker build -t covid-server .
docker run -d --name covid-server --link covid-mongodb -p 9000:9000 covid-server

# Client
cd ../client
docker build -t covid-client .
docker run -d --name covid-client -p 3000:3000 covid-client
```

## üêõ Troubleshooting

### Common Issues

**1. Server won't start**
- ‚úÖ Check MongoDB is running
- ‚úÖ Verify port 9000 is available
- ‚úÖ Check server logs in `logs/` folder

**2. No data displayed**
- ‚úÖ Run manual data update: `POST http://localhost:9000/api/update`
- ‚úÖ Check MongoDB contains data
- ‚úÖ Verify API endpoints return data

**3. Map not loading**
- ‚úÖ Verify Mapbox token is correct
- ‚úÖ Check browser console for errors
- ‚úÖ Ensure token has necessary permissions

**4. CORS errors**
- ‚úÖ Verify client URL in server CORS configuration
- ‚úÖ Check both servers are running on correct ports

### Data Sources

The application uses multiple data sources for reliability:
- Primary: Johns Hopkins CSSE COVID-19 repository
- Fallback: disease.sh API
- Local: Cached data and database
